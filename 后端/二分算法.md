### 循环不变量

循环不变量是循环开始前为真，且在每次循环迭代结束时仍然为真的某个条件或性质。对于二分算法，循环不变量通常涉及以下几个方面：

1. **搜索范围的正确性**：在每次循环开始时，我们知道要查找的元素如果存在，则一定位于当前考虑的数组范围内（即`low`和`high`索引之间，包括`low`和`high`指向的元素）。
2. **中间值的计算**：`mid`（或称为中间索引）的计算是正确的，即它是当前搜索范围（`low`到`high`）的中间位置。
3. **搜索方向的正确性**：根据中间元素与目标元素的比较结果，我们能够正确地决定是继续在左侧半区搜索（即调整`high`指针），还是在右侧半区搜索（即调整`low`指针）。

### 示例说明

以下是一个二分查找算法的伪代码，同时标注了循环不变量的关键部分：

```plaintext
function binarySearch(arr, target):  
    low = 0  
    high = length(arr) - 1  
    while low <= high:  
        // 循环不变量：target 如果在 arr 中，则一定在 arr[low..high] 中  
        mid = low + (high - low) / 2  // 防止溢出，并计算中间索引  
        if arr[mid] == target:  
            return mid  // 找到目标，返回索引  
        else if arr[mid] < target:  
            low = mid + 1  // 排除左侧半区  
        else:  
            high = mid - 1  // 排除右侧半区  
    return -1  // 没有找到目标
```

### 维护和证明循环不变量

- **初始化**：在循环开始前，`low`和`high`分别指向数组的第一个和最后一个元素的索引，因此循环不变量（即目标如果存在，则一定在`arr[low..high]`中）成立。
- **保持**：在每次循环迭代中，我们根据`arr[mid]`与目标值的比较结果来更新`low`或`high`。这些更新确保了如果目标值存在，它仍然位于新的搜索范围内（即新的`arr[low..high]`）。
- **终止**：当`low`超过`high`时，循环终止。此时，如果目标值不存在于数组中，则循环不变量仍然为真（即目标值不在任何非空子数组中），算法返回-1。
